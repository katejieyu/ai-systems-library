Quick Install

Each system is a .md file. Copy it to ~/.claude/commands/ once — then invoke it by name in any Claude Code session.

Product AI System
cp product-ai-system/product-ai-system.md ~/.claude/commands/product-ai-system.md
Portfolio AI System
cp portfolio-ai-system/portfolio-ai-system.md ~/.claude/commands/portfolio-ai-system.md

Product AI System
For designers, PMs, and engineers
Live

Takes a product problem — vague idea, named feature, or draft PRD — and moves it through a structured 9-skill chain to execution-ready output. The system doesn't agree with your framing. It finds the real problem behind your request, challenges the solution before you build it, and breaks everything into tasks with acceptance criteria and a readiness score.

Also has a self-evolution mode — when its own assumptions drift, it can turn inward, detect gaps, and fix its own workflow without a full rebuild.

Define Issue Diagnose Define Goal Generate Solution Challenge Refine Self Study Self Test Execution Readiness ✦ Self-Evolution Mode
/product-ai-system [problem] Full 9-skill cycle — raw problem to execution plan
/product-ai-system fast [problem] Compressed 5-skill pass — faster, less depth
/product-ai-system brief [problem] Problem definition only (Skills 1–2)
/product-ai-system prd [problem] Full PRD output (Skills 1–6)
/product-ai-system skill [1-9] [input] Run a single skill in isolation
/product-ai-system evolve Self-evolution loop — detect drift, fix gaps, re-test
/product-ai-system feature-update [feature] Re-align system after a feature changes
/product-ai-system refresh-persona Update persona and journey definitions
/product-ai-system sync-readme Sync documentation to current system state
Install cp product-ai-system/product-ai-system.md ~/.claude/commands/product-ai-system.md

Portfolio AI System
For designers and PMs applying at Staff / Principal level
Live

Evaluates a portfolio the way a hiring manager actually reads it — not "looks good," but a structured pass across positioning, content signal density, UX, mobile, and copy quality. Runs a 7-step audit and produces exact before/after rewrites. Includes a hiring simulation that gives you a verdict from four perspectives: Recruiter, Hiring Manager, Design Leader, and AI/Product Leader.

Modular skills can be run independently — hire simulation only, mobile QA only, full-site consistency scan, or execution mode that applies all fixes directly to files.

Define Intent Diagnose Issues Goal Alignment Hiring Simulation UX / QA / Mobile Rewrite Execution Plan
/portfolio-ai-system audit [portfolio] Full 7-step audit — positioning, hiring sim, UX, rewrites, execution plan
/portfolio-ai-system scan-site [site] Full cross-page audit — consistency, system-level issues, top 10 fixes
/portfolio-ai-system scan-site + execution [site] Cross-page audit + apply all fixes page-by-page
/portfolio-ai-system content-strategy [portfolio] Define what to say, what to remove, signal hierarchy, tone
/portfolio-ai-system strategy-check [site] Validate site against its defined strategy — alignment score, gaps, top fixes
/portfolio-ai-system learn-site [pages] Extract consistency baseline — positioning, tone, structure, CTA patterns
/portfolio-ai-system consistency-check [page] Check a new or changed page against the established site patterns
/portfolio-ai-system rewrite [section] Rewrite a specific section — headline, CTA, copy block
/hiring-simulation-skill [portfolio] Hiring simulation only — 4-perspective verdicts
/mobile-qa-skill [page] Mobile QA — 375px layout, overflow, tap targets checklist
/portfolio-execution-skill [audit output] Apply all audit fixes directly to files — produces change log
