# Product AI System

A self-evolving AI-native product development system.

## Two Operating Modes

**Product Problem Mode** — analyze a problem, generate a solution, prepare for execution
**Self-Evolution Mode** — evolve the system itself when it drifts, gaps appear, or features change

## Core Loop (Product Problem)
Define Issue → Diagnose → Define Goal → Generate Solution → Challenge → Refine → Self Study → Self Test → Execution Readiness

## Core Loop (Self-Evolution)
Self Inquiry → Hypothesis → Self Test → Gap Detection → Self-Healing Execution → Re-test → (repeat until no new gaps)

## Commands

**Product Problem Mode:**
- `run [problem]` — full 9-skill cycle
- `fast [problem]` — compressed 5-skill cycle
- `brief [problem]` — problem brief only (skills 1–2)
- `prd [problem]` — PRD (skills 1–6)

**Self-Evolution Mode:**
- `evolve` — run self-evolution loop
- `feature-update [feature]` — re-align system after a feature changes
- `refresh-persona` — update persona and journey definitions
- `sync-readme` — sync documentation to current system state

## Workflows
- **product-full-cycle-workflow** — full 9-skill product problem cycle
- **product-self-evolution-workflow** — self-inquiry → hypothesis → test → gap → fix → re-test
- **feature-change-loop-workflow** — understand → impact → update → re-evolve → refresh persona → sync readme

## Sub-Skills (Self-Evolution)
- `self-inquiry` — challenge current assumptions
- `self-hypothesis` — generate testable hypotheses from gaps
- `self-test-loop` — simulate usage, find failure points
- `gap-detection` — find missing logic and undefined states
- `self-healing-execution` — fix gaps directly
- `persona-journey-refresh` — update persona and journey after change
- `readme-sync` — align documentation with current system state

## Operating Principle

AI is not a tool — it is a system.
